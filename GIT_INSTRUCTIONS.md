# Инструкции по созданию коммитов для каждого этапа

## Подготовка репозитория

```bash
cd "c:\Users\User\Desktop\ассемблер_конфигурационка"
git init
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

## Создание .gitignore

```bash
echo "*.bin" > .gitignore
echo "*.pyc" >> .gitignore
echo "__pycache__/" >> .gitignore
echo ".vscode/" >> .gitignore
echo "*.xml" >> .gitignore
echo "analyze_*.py" >> .gitignore
```

## Коммиты для каждого этапа

### Этап 1: Промежуточное представление

```bash
git add README.md assembler.py tests/test_spec.asm
git commit -m "Этап 1: Реализован ассемблер с промежуточным представлением

- Создан язык ассемблера с мнемониками LOAD_CONST, READ_MEM, WRITE_MEM, GTE
- Реализован разбор текстовых команд
- Реализована трансляция в промежуточное представление
- Добавлен тестовый файл со всеми примерами из спецификации
- В режиме --test выводится промежуточное представление"
```

### Этап 2: Машинный код

```bash
git add assembler.py
git commit -m "Этап 2: Реализована генерация машинного кода

- Добавлено преобразование промежуточного представления в байтовый код
- Реализована запись бинарных файлов
- Поддержка little-endian формата
- Вывод размера выходного файла
- В тестовом режиме выводятся байтовые последовательности
- Проверено соответствие тестовым последовательностям из спецификации"
```

### Этап 3: Интерпретатор и память

```bash
git add interpreter.py tests/test_copy.asm
git commit -m "Этап 3: Реализован интерпретатор с моделью памяти

- Создана виртуальная машина с разделённой памятью команд и данных
- Реализован основной цикл интерпретатора
- Реализовано выполнение команд LOAD_CONST, READ_MEM, WRITE_MEM
- Добавлена генерация XML дампов памяти
- Создана тестовая программа копирования массива
- Проверена корректность работы с памятью"
```

### Этап 4: АЛУ и операция >=

```bash
git add interpreter.py tests/test_gte.asm
git commit -m "Этап 4: Реализована команда >= (АЛУ)

- Добавлено выполнение команды GTE (>=)
- Реализована бинарная операция сравнения
- Создана тестовая программа для проверки операции >=
- Проверены различные случаи сравнения: больше, равно, меньше
- Результаты сохраняются в памяти (0 или 1)"
```

### Этап 5: Тестовая задача

```bash
git add tests/test_vectors_*.asm run_tests.ps1
git commit -m "Этап 5: Реализованы программы для тестовой задачи

- Созданы 3 программы поэлементного применения >= к векторам длины 7
- Программа 1: различные значения в векторах
- Программа 2: числа от 50 до 100
- Программа 3: сравнение с константным вектором (все 7)
- Добавлен скрипт run_tests.ps1 для автоматического запуска всех тестов
- Проверена корректность результатов через XML дампы памяти
- Результат записывается во второй вектор"
```

## Финальный коммит

```bash
git add .
git commit -m "Финальная версия: все этапы выполнены

- Полностью реализован ассемблер для УВМ (вариант 15)
- Полностью реализован интерпретатор
- Все команды протестированы
- Документация в README.md
- Тестовые программы для всех этапов"
```

## Проверка истории коммитов

```bash
git log --oneline
```

## Создание тегов для этапов

```bash
git tag -a v1.0-stage1 <commit-hash> -m "Этап 1: Промежуточное представление"
git tag -a v1.0-stage2 <commit-hash> -m "Этап 2: Машинный код"
git tag -a v1.0-stage3 <commit-hash> -m "Этап 3: Интерпретатор"
git tag -a v1.0-stage4 <commit-hash> -m "Этап 4: АЛУ"
git tag -a v1.0-stage5 <commit-hash> -m "Этап 5: Тестовая задача"
```

## Отправка в удалённый репозиторий

```bash
git remote add origin <url-репозитория>
git push -u origin main
git push --tags
```
